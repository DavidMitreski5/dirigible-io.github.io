Integrate Legacy Components

Variety of integration patterns for your legacy components are possible. Java libraries and their APIs can be available from within the scripting services once they are bundled along with the standard Dirigible's runtime package (WAR) using the build framework of your choice e.g. Maven. non-Java libraries and services can be connected either as external processes or as 3-thd party services in integration routes.

Adaptable Software

Based on the In-System Programming Model and accessible worldwide directly via a Browser only, the developers using Dirigible have the freedom and power to follow business requirements just right away. The applications built with the cloud toolkit complies with the Dynamic Applications paradigm, which means they are built for change.

Database Development

Modeling of data structures artifacts in JSON format as well as creation of actual tables and views during activation/publishing

Server-Side Development

Using of most popular dynamic languages such as JavaScript, Ruby and Groovy, for creating server-side services. Support of the standard CommonJS modularization. Generation of fully compliant RESTful services including meta-data support on existing database artifacts.

Client-Side Development

Built-in support for user interface generation for the above RESTful services, based on widely used frameworks such as Bootstrap, jQuery, AngularJS, OpenUI5.
WYSIWYG editor for standard HTML5 as well as for Bootstrap based complex controls.
Wiki scripting support via Confluence format.

Connecting with External Services

Using built-in micro-ESB based on Apache Camel and CXF for declarative integration and orchestration of the business process flows.

Extension Points and Extensions

Declarative definition of extension Points and extensions, for better adaptability as well as better visibility of dependencies and simplified life-cycle management.
